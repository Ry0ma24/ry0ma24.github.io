kubernetes-goat靶机搭建

发表于 2026-04-13 分类于 云安全 阅读次数：

环境

应用 版本
RockyLinux 10.1
Docker 29.4.0
minikube 1.38.1
helm 4.1.4

docker

单机版的K8s 需要用docker

[root@localhost ~]# dnf -y install dnf-plugins-core git
[root@localhost ~]# dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
[root@localhost ~]# dnf install docker-ce docker-ce-cli  docker-buildx-plugin docker-compose-plugin
[root@localhost ~]# systemctl start docker 
[root@localhost ~]# docker info
Client: Docker Engine - Community
 Version:    29.4.0
 Context:    default
 Debug Mode: false
 Plugins:
<...>
[root@localhost ~]# systemctl enable docker  //设置开机自启

minikube

安装minikube

[root@localhost ~]# curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-latest.x86_64.rpm
[root@localhost ~]# rpm -Uvh minikube-latest.x86_64.rpm

启动minikube

配置kubectl

kubernetes-goat 需要使用kubectl，但是minikube使用 kubectl 命令为 minikube kubectl ，做个软链接即可

[root@localhost kubernetes-goat]# ln -s $(which minikube) /usr/local/bin/kubectl

问题解决

[root@localhost ~]# cd /root/kubernetes-goat/scenarios/health-check
[root@localhost ~]# sed -i '/socket-link/d' deployment.yaml 
[root@localhost kubernetes-goat]# helm list --all-namespaces
NAME            NAMESPACE       REVISION        UPDATED                                 STATUS      CHART                    APP VERSION
metadata-db     default         1               2026-04-12 01:42:54.41527795 +0800 CST  deployed    metadata-db-0.1.0        1.0        
[root@localhost kubernetes-goat]# helm uninstall metadata-db //卸载残留文件
release "metadata-db" uninstalled
